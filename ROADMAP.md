# 栈问 - 开发计划

## 项目概述

栈问是一个生活经验问答社区，用户可以提问、回答、点赞、采纳答案，通过积分系统激励优质内容产出。

## 已完成功能 ✅

### 1. 用户系统
- [x] 用户注册、登录、登出
- [x] 个人资料页（支持查看他人资料）
- [x] 个人设置（修改用户名、邮箱、密码）
- [x] 用户权限管理（普通用户、管理员）
- [x] 用户停用功能

### 2. 问答系统
- [x] 发布问题

- [x] 问题浏览（热度、最新、未解决排序）
- [x] 问题搜索（标题、正文、作者）
- [x] 相关问题推荐
- [x] 问题浏览量统计

### 3. 积分系统

## 技术栈

### 前端
- **框架**：Next.js 15 (App Router)
- **语言**：TypeScript
- **样式**：CSS Modules / 全局 CSS
- **UI 组件**：自定义组件
- **图标**：Lucide React
- **富文本编辑器**：Tiptap
- **代码高亮**：Lowlight

### 后端
- **框架**：Next.js API Routes / Server Actions
- **数据库**：SQLite (开发) / PostgreSQL (生产推荐)
- **ORM**：Prisma
- **认证**：自定义 Session (基于 Cookie)
- **密码加密**：bcrypt

### 部署
- **平台**：Vercel (推荐) / 自托管
- **数据库**：Vercel Postgres / Supabase / 自托管

## 开发规范

### 代码规范
- 使用 TypeScript 严格模式
- 使用 ESLint 和 Prettier 格式化代码
- 组件使用函数式组件
- 使用 Server Components 优先，Client Components 按需使用

### Git 规范
- 提交信息格式：`<type>: <description>`
  - `feat`: 新功能
  - `fix`: 修复 bug
  - `refactor`: 重构
  - `style`: 样式调整
  - `docs`: 文档更新
  - `test`: 测试相关
  - `chore`: 构建/工具相关

### 数据库规范
- 使用 Prisma Migration 管理数据库变更
- 每次 schema 变更都要创建 migration
- 生产环境谨慎执行 migration

## 部署指南

### 开发环境
```bash
# 安装依赖
npm install

# 初始化数据库
npx prisma migrate dev

# 启动开发服务器
npm run dev
```

### 生产环境
```bash
# 构建
npm run build

# 启动生产服务器
npm start
```

### Vercel 部署
1. 连接 GitHub 仓库
2. 配置环境变量
3. 自动部署
