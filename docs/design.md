# 栈问社区设计说明

## 设计来源

UI 以 `preview/` 中的静态页面为基准迁移。视觉方向保持轻量产品工具：稳定、清晰、克制，强调列表扫描、状态辨识和表单效率。

## 视觉原则

- 使用浅色产品界面，适合用户在白天办公或学习环境中持续浏览问答。
- 色彩采用冷调中性底色，绿色作为主操作和成功状态，蓝色用于标签信息，黄色用于未解决状态。
- 卡片半径保持 8px 左右，避免过度装饰。
- 首页第一屏直接给问题列表，不做营销落地页。
- 交互控件保持标准：搜索框、分段排序、标签筛选、按钮、表单、状态徽标。

## 组件约定

- 顶部导航固定在页面上方。
- 问题列表用左侧指标列加正文摘要。
- 详情页突出问题、回答、点赞和采纳操作。
- 表单错误用行内提示，不用弹窗。
- 积分使用等宽数字，便于扫描。
